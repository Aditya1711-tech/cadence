## 1. What we are building

Cadence is a **privacy-first activity and time tracker for software teams**. It
runs quietly in the background, understands what a developer actually spent time
on across their editor, browser, terminal, meetings, and AI coding tools, and
turns that into useful personal insight and team-level analytics.

The wedge that no competitor has: **it tracks AI coding-agent token spend**
(Claude Code, Codex, Cursor, etc.) as a first-class signal, alongside time and
git activity.

### Who it is for (first)
B2B, developer teams of 5–50 people. The buyer is a technical lead or founder
who installs it, sees value, and rolls it to the team.

### What makes it different
- **Local-first.** Raw data lives on the user's machine in an encrypted store.
  Only what the user opts to share leaves the device.
- **No screenshots, ever.** This is a deliberate identity choice. We are
  trust-first, not surveillance-first.
- **AI cost visibility.** Token burn per developer, per model, per session.

### What we are explicitly NOT building (v1)
Mobile apps. Non-developer collectors (Figma, Salesforce). Screen recording or
screenshots. Open-ended AI chat. Keep these out of scope; they are later bets.

## 2. Glossary (use these words consistently)

| Term | Meaning |
|---|---|
| **Collector** | Anything that produces raw activity events: OS daemon, VSCode ext, Chrome ext, token watcher, GitHub poller. |
| **Agent / Daemon** | The local background process that aggregates collectors, stores events locally, applies privacy rules, and syncs. |
| **Event** | One atomic activity record (see the Event Contract below). |
| **Category** | A semantic label on an event: `deep_work`, `meetings`, `comms`, `research`, `idle`, etc. |
| **Org** | A paying tenant. Contains members and teams. |
| **Seat** | One member's paid license within an org. |
| **Spine** | The one stream in a phase that must finish before the others can parallelize. |
| **Stream** | A unit of parallel work = one Claude Code session. IDs like `P1-A`. |
| **Wave** | A set of streams that can run at the same time. |

## 5. THE EVENT CONTRACT (frozen)

Every collector emits events in **exactly** this shape. This is the single most
important contract in the system — it is what lets the daemon, the extensions,
and the backend be built in parallel.

```jsonc
{
  "event_id":    "uuid-v4",          // generated by the collector
  "schema_ver":  1,                  // bump only via coordination event
  "source":      "vscode",           // enum: os | vscode | chrome | token | github
  "member_id":   "uuid",             // local member identity (set at install)
  "ts_start":    "2025-06-01T09:14:02Z",  // RFC3339 UTC
  "ts_end":      "2025-06-01T09:18:45Z",  // RFC3339 UTC
  "duration_ms": 283000,             // ts_end - ts_start, precomputed
  "app":         "Visual Studio Code",     // human-readable app name
  "title":       "auth.ts — cadence-api",  // window/tab title, may be redacted
  "url":         null,               // present for chrome source only
  "project":     "cadence-api",      // best-effort project/repo name, nullable
  "category":    "deep_work",        // set by rule classifier; nullable pre-classify
  "is_idle":     false,              // true if no input during the window
  "meta": {                          // source-specific, additive only
    "lang":      "typescript",       // vscode
    "model":     "claude-sonnet-4",  // token source
    "tokens_in":  12000,             // token source
    "tokens_out": 3400,              // token source
    "cost_usd":   0.045,             // token source
    "commit_sha": "abc1234",         // github source
    "repo":       "org/cadence-api"  // github source
  }
}
```

**Rules for the contract:**
- Unknown `meta` keys are ignored, never rejected. `meta` is additive forever.
- `category` enum (v1): `deep_work`, `meetings`, `comms`, `research`,
  `code_review`, `ai_assisted`, `idle`, `other`.
- Timestamps are always UTC RFC3339. The UI localizes; storage never does.
- A collector that cannot fill a field sends `null`, never omits the key.

## 6. REST API CONVENTIONS (frozen)

Base path: `/api/v1`. JSON only. Auth via `Authorization: Bearer <jwt>`.

| Method | Path | Purpose | Stream that owns it |
|---|---|---|---|
| `POST` | `/api/v1/ingest/events` | Batch event upload from daemon | P2-A (defines), P2-B (client) |
| `GET`  | `/api/v1/me/timeline?from&to` | Personal timeline | P2-A |
| `GET`  | `/api/v1/me/summary?range` | Personal rollups | P2-A |
| `GET`  | `/api/v1/org/members` | Org roster (admin) | P2-E |
| `GET`  | `/api/v1/org/summary?range&team` | Team rollups (admin) | P2-E |
| `POST` | `/api/v1/auth/login` | Issue JWT | P2-A |
| `POST` | `/api/v1/auth/register-org` | Create org + first admin | P2-A |
| `GET`  | `/api/v1/insights/weekly` | Weekly narrative | P3-A |
| `POST` | `/api/v1/query/nl` | Natural-language query | P3-C |
| `POST` | `/api/v1/billing/webhook` | Stripe webhook | P3-D |

**Conventions:**
- Ingest accepts an array, max 1000 events per call, idempotent on `event_id`.
- All list endpoints paginate with `?cursor=` + `?limit=` (default 100).
- Errors use RFC 7807 problem+json: `{ "type", "title", "status", "detail" }`.
- Every privacy-sensitive field obeys the org's privacy policy server-side
  (see §8) — never trust the client to have redacted.

## 7. DATABASE CONVENTIONS (frozen)

- Postgres 16. The `events` table is a **TimescaleDB hypertable** partitioned on
  `ts_start`. Continuous aggregates produce hourly + daily rollups.
- All tables have `id uuid primary key default gen_random_uuid()`,
  `created_at timestamptz not null default now()`.
- Tenancy: every row that belongs to an org carries `org_id uuid not null`.
  Row-level security enforced; queries always filter by `org_id`.
- The async work queue is a table, not SQS:

```sql
create table job_queue (
  id           uuid primary key default gen_random_uuid(),
  kind         text not null,            -- 'categorize' | 'digest' | ...
  payload      jsonb not null,
  status       text not null default 'pending', -- pending|running|done|failed
  attempts     int  not null default 0,
  run_after    timestamptz not null default now(),
  locked_by    text,
  locked_at    timestamptz,
  created_at   timestamptz not null default now()
);
create index on job_queue (status, run_after);
```

Workers claim jobs with `SELECT ... FOR UPDATE SKIP LOCKED`. This gives us SQS
semantics with zero extra AWS services.

- Migrations: Flyway, versioned `V1__init.sql`, `V2__...`. Owned by the spine
  stream of each phase. Other streams **request** a migration via the Build Log;
  they do not write to the migrations folder.

## 8. PRIVACY MODEL (frozen)

The privacy policy is per-org and applied **server-side on ingest** and again on
read. Levels:

| Level | What the org admin sees |
|---|---|
| `full` | App names, window titles, URLs, projects. |
| `categories_only` | Categories + durations + projects. No titles/URLs. |
| `aggregate_only` | Daily totals per category. No per-event detail. |

- The daemon also has a **local redaction list** (regex) the user controls;
  matching titles/URLs are hashed before they ever leave the device.
- GitHub integration has a hard toggle: `commit_messages_only` (default) vs
  `full_diff` (opt-in). Never default to reading code.
- Token events never include prompt/response content — only counts and cost.

## 9. REPO LAYOUT + DIRECTORY OWNERSHIP (frozen)

Monorepo. Each stream owns disjoint directories so parallel sessions never edit
the same file.

```
/docs/                 <- these planning files (everyone reads, spine edits)
/agent/                <- Go daemon + local store + rule classifier   [P1-A]
/ext-vscode/           <- VSCode extension                            [P1-B]
/ext-chrome/           <- Chrome extension                            [P1-C]
/web/                  <- Next.js app (shared shell)                  [spine of P1/P2]
  /web/dashboard/      <- personal dashboard                          [P1-D]
  /web/admin/          <- org admin                                   [P2-E]
  /web/insights/       <- weekly digest UI, NL query UI               [P3-A/P3-C UI]
/backend/              <- Spring Boot                                 [P2-A spine]
  /backend/ingest/     <- ingest API                                  [P2-A]
  /backend/query/      <- query API                                   [P2-A]
  /backend/worker/     <- job worker + categorisation                 [P2-F]
  /backend/token/      <- token watcher ingestion glue                [P2-C]
  /backend/github/     <- github integration                          [P2-D]
  /backend/insights/   <- digest + pattern + NL query                 [P3-A/B/C]
  /backend/billing/    <- Stripe                                      [P3-D]
  /backend/migrations/ <- Flyway SQL (PHASE SPINE ONLY)               [spine]
/deploy/               <- docker-compose, EC2 scripts                 [spine]
```

**Ownership rule:** if a task needs a file outside your stream's directories,
do not edit it. Write a "needs" line in the Build Log requesting it from the
owning stream (or the spine).
